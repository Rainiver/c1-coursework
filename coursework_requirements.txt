Preamble
Your goal is to build a full-stack, research-grade system capable of learning and serving
neural-network models that can interpolate any 5-dimensional numerical dataset. A test
dataset is provided.
The system must have 6 components:
1. APython packageimplementing data loading, model training, and inference.
2. AReadTheDocshtml build (local).
3. AFastAPI backendexposing REST endpoints for training and prediction.
4. ANext.js frontendproviding an interactive UI to train and query models.
5. AGitLabrepository containing all of the above. The commit history should
reflect your answers to each of the questions (use a different tag for each question).
6. ADocker-composefile to serve the app in a container.

appropriately. Implement functionality to split data into train/validation/test sets and
standardizes features. Your implementation should handle the test dataset provided.

4 Implement model [15]
Implement a lightweight neural network torch, scikit-learn or tensorflow. The
model should use small hidden layers (e.g., [64, 32, 16]) and be fully configurable by
number of layers, neurons per layer, learning rate, and max iterations. Performance is
critical: the model must train on CPU within less than a minute for datasets up to 10,000
samples. Ensure the implementation provides a consistentfit(X, y) andpredict(X)
interface compatible with torch/tensorflow conventions. Your implementation should
work with the dataset provided.
5 FastAPI backend [20]
Create a comprehensive FastAPI backend that handles dataset upload and model
operations. The API should provide aGET /health endpoint for service health checks,
aPOST /upload endpoint for uploading 5D datasets in.npz format, and aPOST
/train endpoint that trains models on uploaded datasets. Additionally, implement a
POST /predict for making predictions on new 5D input vectors.
6 Next.js frontend [20]
Develop a modern, responsive Next.js frontend that provides an intuitive interface
for the machine learning system. The application should feature four main pages: an
upload page (/upload) with a file upload interface for.npz datasets including
validation and preview functionality, a training button (/train) where users can
configure hyperparameters and start training, a prediction button (/predict) with an
interactive form containing 5 input fields for making predictions. The result of the
prediction should display.
7 Testing and reproducibility [20]
Create a comprehensive testing suite underbackend/tests/ that validates all
system components and ensures reliability. Additionally, implement containerization
usingDockerfile configurations and adocker-compose.yml file that can launch both
backend and frontend services seamlessly. The implementation should include thorough
documentationofallenvironmentvariables,usageinstructions,andsetupproceduresina
detailed README.md file that enables easy deployment and development workflow.
8 Performance and profiling [20]
Conduct comprehensive performance analysis and benchmarking of the model to
understanditscomputationalcharacteristicsandscalability. Thisanalysisshouldmeasure
training time as a function of dataset size using datasets of 1K, 5K, and 10K samples to
establish scaling behavior. Additionally, profile memory usage during both training and
prediction phasesto identifypotential bottlenecks, and compareaccuracy metricssuch as
MSE and R² scores across different dataset sizes. You should report on your findings in
the documentation page (next question), in a section called "Performances and profiling".
